# M2 截图选区 + 标注 — 需求规格

> 任务: m2-capture-annotate · 策略: full-collaborate · Phase 1 研究与分析产物
> 对照 design/BRIEF.md 界面①、design/mockup.html(L454–554)、design/PLAN.md 里程碑 M2。

## 目标
按 F1(或托盘「截屏」)触发全屏取屏,进入一个覆盖全屏的半透明遮罩;用户拖拽出矩形选区,
在选区右下角外侧弹出经典两排工具栏,提供**全套标注工具 + 撤销/重做 + 复制/保存/确认/取消**。
确认后把(选区原图 + 标注)合成的位图复制到剪贴板(并可保存为文件)。
替换 `app_context._placeholder_feature("screenshot")` 占位,接上真实截图流程。

## 范围(本轮仅 M2)

**IN(本轮交付):**
- **全屏取屏**:多显示器整桌面截屏(mss 抓帧 → QImage/QPixmap),进入全屏无边框置顶遮罩窗。
- **选区交互**:
  - 选区外半透明黑遮罩 `rgba(0,0,0,0.45)`,选区内显示原始清晰画面。
  - 拖拽创建选区;8 个白色控制点(四角 + 四边中点)可调整大小;选区内部可整体拖动。
  - 选区左上角实时显示尺寸 `宽 × 高`。
  - 全屏放大镜 + 取色:像素网格放大 + 当前坐标 `(x, y)` + 颜色 HEX(选区未定时跟随鼠标)。
- **标注工具(上排)**:矩形、椭圆、直线、箭头、画笔(自由曲线)、马克笔(半透明荧光)、
  马赛克(像素化)、文字、序号标记(①②③自动递增)、橡皮擦(删除整个标注对象)。
- **二级属性条(上下文浮层)**:仅选中绘图工具时显示;线条粗细(细/中/粗)+ 颜色色板(8 色)。
- **操作栏(下排)**:撤销、重做 | 取色 | 钉图、复制、保存、**文字识别(OCR)** | 取消(×)、确认(✓)。
- **撤销/重做**:标注对象级别的 undo/redo 栈(Ctrl+Z / Ctrl+Y)。
- **复制 / 保存**:合成「选区原图 + 标注层」→ 复制到系统剪贴板;保存为 PNG/JPG(走 config.screenshot 设置)。
- **快捷键**:Esc 取消、Enter 确认、Ctrl+Z/Y 撤销重做。

**OUT(后续里程碑,本轮做占位按钮但不实现):**
- **钉图**按钮 → M3(本轮点了打日志/占位)。
- **文字识别(OCR)**按钮 → M4(本轮点了打日志/占位)。
- 贴图浮窗、右键菜单 → M3。
- OCR 引擎与结果窗 → M4。
- PyInstaller 打包 → M5。

## 约束
- 框架:**PySide6 (Qt6)**;新增依赖 **mss**(跨平台截屏)。
- 目标平台:**Windows 10**;开发机为 **Linux + Python 3.14 + headless + 无 pip** → 本机仅 `py_compile` + 代码审查,真实运行/外观/截屏在 **Windows 实测**。
- **全中文 UI**(界面文案 + 注释中文)。
- 视觉遵循已有 design token(`ui/theme/tokens.py`:主色 `#2D7FF9`、圆角 6/8/12、柔阴影);
  工具栏/控制点/遮罩样式须还原 mockup 界面①。
- 完全离线,无外部依赖下载;架构须与 M1 一致(信号驱动、core 零 GUI 依赖、UI 可扩展)。
- 与 M1 集成:复用 `app_context` 协调器、`config.settings`(screenshot 段)、`theme`、`logger`。
- 性能:全屏遮罩 + 实时放大镜/绘制须流畅,避免每帧全图重绘卡顿(脏区/缓存策略)。

## 验收标准
1. 按 F1 / 托盘「截屏」→ 弹出全屏遮罩,可拖拽出选区,8 控制点可改大小,选区可整体移动。
2. 选区左上角尺寸实时更新;放大镜显示坐标 + HEX 颜色。
3. 上排 10 个标注工具均可绘制;选中绘图工具时属性条出现,可改粗细 / 颜色;未选时属性条隐藏。
4. 序号标记自动递增;马赛克对选区像素生效;橡皮擦可删除标注对象。
5. 撤销/重做对标注对象生效(含快捷键)。
6. 确认 → 选区+标注合成图进入剪贴板;保存 → 按配置格式写入文件;取消/Esc → 关闭遮罩无残留。
7. 钉图、OCR 按钮存在且可点击(占位打日志),不报错。
8. 代码结构清晰、模块可扩展、命名语义化;关键交互与异常有日志;无 GUI 依赖泄漏到 core。

## 跨平台现实
截屏(mss)、全屏遮罩、剪贴板写图的真实行为强依赖 Windows 桌面环境。
本机 headless 无法显示窗口/抓屏 → M2 在 Linux 侧验证「能 py_compile + 纯逻辑单测(几何/undo 栈/颜色解析/序号递增)+ 代码审查」;
真实截屏/遮罩/绘制/剪贴板/保存留待 **Windows 10 实测**(README 附 M2 逐条验证清单)。
代码须对 headless / 无屏幕 / mss 不可用做防御性降级(不崩,打日志提示)。
