# M1 项目骨架 — 需求规格

> 任务: m1-skeleton · 策略: full-collaborate · Phase 1 研究与分析产物

## 目标
搭建 SnapOCR(截图精灵)的 PySide6 桌面应用骨架,实现一个可常驻运行的程序外壳:**系统托盘 + 全局热键 + 设置窗**。为后续 M2–M4 功能模块提供稳定、可扩展的工程结构与应用框架。

## 范围(本轮仅 M1)

**IN(本轮交付):**
- PySide6 工程结构:可运行的 app 入口、清晰的模块划分(app / ui / core / config)
- **系统托盘常驻**:托盘图标 + 右键菜单,菜单项对照 mockup ④——截屏 / 贴图 / 从剪贴板贴图 / 文字识别 / 最近文件(子菜单) | 设置 | 开机启动(勾选) | 帮助 / 关于 | 退出
- **全局热键注册框架**:F1 截屏 / F3 贴图 / Shift+F3 隐藏显示贴图 / F4 文字识别;触发时走占位回调(打日志),为 M2+ 预留接入点
- **设置窗**:左侧导航(常规 / 热键 / 截屏 / 贴图 / 输出 / 文字识别 / 关于);热键页可显示并修改快捷键并持久化;底部 恢复默认 / 取消 / 确定
- **配置持久化**:热键映射 + 基础选项(QSettings 或 JSON)
- **应用生命周期**:单实例、最小化到托盘、安全退出

**OUT(后续里程碑,本轮不做):**
- 截图选区 / 标注工具(M2)
- 贴图浮窗 + 右键菜单(M3)
- 本地 OCR 接入 + 结果窗(M4)
- PyInstaller 打包 exe(M5)

## 约束
- 框架:**PySide6 (Qt6)**
- 目标平台:**Windows 10**;开发机为 **Linux**
- **全中文 UI**(界面文案 + 注释中文)
- 视觉遵循 design token:主色 `#2D7FF9`、圆角 6/8/12px、柔和阴影
- 完全离线,无外部 CDN / 账号
- 全局热键:Windows 用 keyboard / Win32 API;**Linux 上需可优雅降级**(不可因热键不支持而崩溃)

## 验收标准
1. 程序能启动并驻留(托盘图标出现,无报错)
2. 托盘右键菜单项齐全、可点击(占位动作打日志)
3. 设置窗能打开;热键页能显示 / 修改 / 保存快捷键,重启后保留
4. 全局热键注册流程不报错;Linux 不支持时优雅降级 + 日志提示
5. 代码结构清晰、模块可扩展、命名语义化、关键交互与异常有日志

## 跨平台现实
全局热键与托盘的真实行为强依赖 Windows。M1 在 Linux 侧验证"**能启动 + UI 正确 + 配置可存取 + 降级不崩**";热键/托盘的真实触发行为留待 Windows 实测(或 M5 打包后)。代码须对平台差异做防御性处理。
