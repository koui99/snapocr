# 开发计划：截图精灵 / SnapOCR

> Windows 10 桌面截图 / 贴图 / 标注工具 + 本地离线 OCR，最终交付单文件 `.exe`，全中文界面。
> 本文给非技术读者看，技术细节由开发负责，你只需确认方向。

---

## 一、技术选型（已为你定好，附理由）

| 部分 | 选用 | 为什么 |
|------|------|--------|
| 界面框架 | **Python + PySide6 (Qt)** | 能高度还原已定稿的现代浅色 UI；做托盘、置顶浮窗、全屏截图遮罩都成熟稳定 |
| OCR 引擎 | **RapidOCR（本地离线，ONNX）** | 中英文识别效果好；模型小（约 10–15MB）；**完全离线、不要账号、永久免费**（开源 MIT） |
| 截图取屏 | **mss + Qt 透明全屏窗口** | 多显示器、像素级取色、放大镜都能做 |
| 全局热键 | **keyboard / Win32 API** | F1 截屏、F3 贴图等后台快捷键 |
| 打包 | **PyInstaller** | 把 Python + 模型 + 依赖压成**单个 exe**，双击即用、免安装 |

**结论**：纯本地、免费、免联网、免安装。代价是 exe 体积偏大（预计 80–130MB，因为内置了识别模型和运行库），这是离线 OCR 的正常代价。

## 二、功能清单（对照已定稿 UI）

### 截图与标注（界面①）
- F1 全屏取屏 → 拖拽选区，半透明遮罩 + 8 控制点 + 实时尺寸
- 放大镜 + 取色（显示坐标 + HEX）
- 标注工具：矩形、椭圆、直线、箭头、画笔、马克笔、马赛克、文字、**序号标记(①②③自动递增)**、橡皮擦
- 颜色/粗细属性条：选中绘图工具时才弹出（上下文浮层，已定）
- 撤销/重做、复制、保存、取消/确认

### 贴图浮窗（界面②）
- F3 把截图/剪贴板图片钉在桌面，无边框置顶浮窗
- 右键菜单：复制、保存、OCR、缩放、不透明度、旋转、置顶、取消贴图、销毁
- 滚轮缩放、拖动移动

### OCR 文字识别（界面③，本地离线）
- 在截图工具栏点「识别」或对贴图右键 → 调本地 RapidOCR
- 弹结果窗：原图缩略 + 可编辑文本框、识别语言下拉、重新识别
- 复制全部 / 保存 txt、显示耗时与置信度
- （翻译按钮先留位，默认不接在线翻译；要的话单独商量）

### 设置 + 托盘（界面④）
- 托盘常驻，右键菜单快捷入口
- 设置页：常规 / 热键(可改) / 截屏 / 贴图 / 输出 / OCR / 关于
- 开机启动、默认保存路径、图片格式等

## 四、一个需要你知道的现实点（关于"生成 exe"）

我当前的开发机是 **Linux**，可以写全部代码、出界面，但**最终把代码打包成 Windows `.exe` 这一步必须在 Windows 环境完成**（PyInstaller 不能在 Linux 上造 Windows exe）。两种无痛方案，你选其一即可（到 M5 再决定）：

- **方案A（推荐，你不用碰代码）**：用 GitHub Actions 的 Windows 云机器自动打包，跑完直接下载 exe。
- **方案B**：我把代码和一行打包命令给你，你在自己的 Win10 上跑一下生成 exe。

开发和功能验证我会在过程中尽量在 Linux 侧完成，截图/热键/托盘这类强依赖 Windows 的部分会在打包后于 Windows 上实测。
