# UI 设计简报：截图标注工具（Snipaste 克隆 + OCR）

## 产品定位
Windows 10 桌面截图/贴图/标注工具，功能对标 Snipaste，额外新增 **OCR 文字识别**。
全中文界面。这一步只做 **静态 UI 布局效果图**（不写真实业务逻辑）。

## 交付物
生成一个**自包含的单文件** `mockup.html`：
- 所有 CSS 内联在 `<style>` 里，禁止引用任何外部 CDN/字体/JS 库（离线可看）。
- 图标用内联 SVG 或 Unicode 字符，不要外链。
- 页面用纵向分区，每个界面一个「展示卡片」，卡片上方有中文标题和一句说明。
- 顶部放一个总标题栏：产品名「截图精灵 / SnapOCR」+ 版本号 + 简短介绍。
- 用一个浅灰画布背景模拟桌面，让浮窗类界面看起来"漂浮"在桌面上。

## 视觉风格：现代浅色
- 背景浅灰 (#F5F6F8 ~ #ECEEF1)，卡片纯白。
- 微圆角：工具栏/按钮 6–8px，窗口 10–12px。
- 柔和阴影：0 4px 16px rgba(0,0,0,0.08) 这类，不要硬黑边。
- 主色调蓝色 (#2D7FF9 左右) 用于选中/强调/主按钮；选区高亮边框用主色。
- 字体用系统中文字体栈（Microsoft YaHei / PingFang / sans-serif）。
- 整体克制、专业，像 Win11 现代应用，不要花哨渐变。

## 需要画的 4 组界面

### 1. 截图选区 + 标注工具栏（核心界面）
模拟一次截图：在一张"桌面截图"上拖出一个矩形选区。
- 选区外区域用半透明黑色遮罩 (rgba(0,0,0,0.45))，选区内清晰。
- 选区四周 8 个白色控制点 + 主色边框。
- 选区**左上角**显示尺寸信息，例如 `1280 × 720`。
- 选区**右下角外侧**贴着两条横向工具条（这是 Snipaste 的经典布局）：
  - **标注工具栏（上排）**：矩形、椭圆、直线、箭头、画笔、马克笔(荧光)、马赛克、文字、序号标记、橡皮擦。每个图标后给中文 tooltip 说明文字（可直接标在图标下方小字）。
  - **操作栏（下排）**：撤销、重做 | 取色 | 钉图(图钉图标)、复制、保存、**文字识别(OCR)**、取消(×)、确认(✓)。把"文字识别"按钮做得稍微醒目一点（它是新功能）。
- 工具栏当前选中"矩形"工具时，下方再画一个**二级属性条**：线条粗细（细/中/粗三档圆点）、颜色色板（红橙黄绿蓝紫黑白几个圆点）。
- 鼠标位置画一个**放大镜+取色**小窗：放大的像素网格 + 当前坐标 + 颜色 HEX 值 `#3A7BD5`。

### 2. 贴图浮窗（Pin）
一个钉在桌面上的小图片窗口：
- 无标题栏的纯图片浮窗，带柔和投影和细边框，看起来浮在桌面上。
- 一角显示缩放比例徽标，例如 `100%`。
- 旁边展开一个**右键菜单**，中文菜单项：复制、保存图片、文字识别(OCR)、缩放(放大/缩小/恢复100%)、不透明度(子菜单箭头)、旋转、置顶、取消贴图、销毁。带分隔线分组。

### 3. OCR 文字识别窗（新功能，重点）
识别完成后弹出的结果面板窗口：
- 标准窗口：标题栏「文字识别结果」+ 右上角最小化/最大化/关闭三个按钮。
- 左右两栏（或上下）：左/上是**原图缩略**（带可框选区域提示），右/下是**可编辑的识别文本框**，里面放几行示例中文+英文混排文本。
- 顶部一行工具：识别语言下拉（简体中文/英文/中英混合）、重新识别按钮。
- 底部操作栏：复制全部、保存为 txt、翻译(可选，做成次要按钮)、关闭。
- 文本框下方状态栏：`识别耗时 0.8s · 置信度 96%`。

### 4. 设置窗 + 托盘右键菜单
**设置窗**：经典左侧导航 + 右侧内容布局。
- 左侧竖直导航项：常规、热键、截屏、贴图、输出、文字识别(OCR)、关于。当前高亮"热键"。
- 右侧"热键"页内容：几行「功能名 + 快捷键输入框」，例如：截屏 `F1`、贴图 `F3`、隐藏/显示贴图 `Shift+F3`、文字识别 `F4`。每行右侧一个"修改"提示。
- 窗口底部：恢复默认、取消、确定 按钮。

**托盘右键菜单**（单独画一个小卡片，模拟从 Windows 右下角任务栏托盘弹出）：
- 菜单项：截屏、贴图、从剪贴板贴图、文字识别、最近文件(子菜单箭头) | 设置 | 开机启动(带勾选) | 帮助、关于 | 退出。

## 布局排版要求
- 4 组界面从上到下排列，界面 1 占满宽度，其余可两列并排（OCR 窗 + 贴图浮窗一行，设置窗 + 托盘菜单一行）以节省纵向空间，但要保证每个都看得清。
- 每个展示卡片配中文标题序号（① 截图选区与标注 …）。
- 收尾放一个小的**设计说明区**：列出用到的主色、圆角、阴影规范（设计 token），方便后续开发还原。
- 代码整洁、语义化 class 命名，方便我后续直接拿去改成真实前端。
