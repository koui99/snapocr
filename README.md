# 截图精灵 / SnapOCR

Windows 10 桌面**截图 / 贴图 / 标注 + 本地离线 OCR** 工具,全中文界面,基于 PySide6 实现,最终以 PyInstaller 打包为单个 `.exe`。

> **当前进度:M4 OCR 文字识别** —— 本地离线 RapidOCR + 结果窗(语言下拉/重新识别/可编辑文本/复制·存txt/耗时·置信度)。
> 已具备:M1 托盘/热键/设置窗、M2 截图选区+标注、M3 贴图浮窗。
> 打包(M5)为后续里程碑。

## 技术栈

| 部分 | 选用 |
|------|------|
| 界面 | PySide6 (Qt6) |
| 全局热键 | Windows: `ctypes` 直调 Win32 `RegisterHotKey`;非 Windows:自动降级(仅记录,不崩) |
| 配置持久化 | 自管 JSON(`%APPDATA%/SnapOCR/config.json`) |
| 单实例 | QLocalServer / QLocalSocket(可唤醒已有实例) |
| 打包(M5) | PyInstaller 单 exe |

## 运行(Windows 10)

```bat
py -3.12 -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python run.py
```

> ⚠️ PySide6 不一定支持最新的 Python 3.14,建议使用 **Python 3.11–3.13**(本项目用 3.12 验证)。

**入口与识别**
- [ ] 截图选区 → 点工具栏「识别」→ 弹出「文字识别结果」窗,文本框出现识别文字

- [ ] 旋转过的贴图点「文字识别」→ 退回弹独立结果窗(旋转态不原地叠加)
- [ ] 复制一张带文字的图 → F4 / 托盘「文字识别」→ 弹结果窗识别剪贴板图片
- [ ] 剪贴板无图时按 F4 → 提示「剪贴板没有图片,无法识别文字」,不崩

**结果窗(界面③)**
- [ ] 左侧原图缩略正确(等比缩放、不变形)
- [ ] 右侧文本框可编辑;识别中状态栏显示「识别中…」且 UI 不冻结(可拖动窗口)
- [ ] 状态栏显示「识别耗时 X · 置信度 Y% · N 行」
- [ ] 语言下拉切换(中英混合/仅英文)→ 自动重新识别;「重新识别」按钮可重跑
- [ ] 复制全部 → 文本进剪贴板,可在别处粘贴
- [ ] 保存 txt → 弹保存框,写出 UTF-8 文本文件,状态栏提示文件名
- [ ] 翻译按钮存在但禁用(灰色,悬停提示「默认不接在线翻译」)
- [ ] 底部有「关闭」按钮 → 点击关闭结果窗(标题栏 ✕ 同样可关)

**异常与降级**
- [ ] 未安装 rapidocr 时点识别 → 状态栏提示「OCR 组件未安装…」,程序不崩
- [ ] 识别不到文字的图 → 状态栏提示「未识别到文字」,不崩
- [ ] 连续多次「重新识别」/ 多个结果窗并存 → 不崩、不卡死(并发已加锁)
- [ ] 识别进行中直接关闭结果窗 → 不闪退(QThread 已做安全处理)

**设置**
- [ ] 设置窗「文字识别(OCR)」页:默认识别语言下拉 + 自动复制开关,保存后生效
- [ ] 开「自动复制」后识别完,文本已在剪贴板(状态栏仍显示耗时/置信度)

## 开发说明(Linux / headless)

- 开发机为 Linux 且无显示环境时,无法显示真实窗口。可用 offscreen 平台做导入/启动冒烟:
  ```bash
  QT_QPA_PLATFORM=offscreen python run.py
  ```
- `core` 层除热键门面借用 Qt 信号外,不依赖任何窗口控件,可独立单测。
- 视觉规范(主色 `#2D7FF9`、圆角、阴影、字体)集中在 `src/ui/theme/tokens.py`,改样式只改这一处。
- 真实的托盘行为、全局热键、窗口外观需在 **Windows 实测**。
